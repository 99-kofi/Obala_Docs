1. Base URL

2. Authentication Overview

OBALA uses API keys for authentication. Each user must sign up to generate their unique key.

Type Required Description
X-API-Key ‚úÖ Yes Sent in headers to authenticate your requests

3. Create an Account (Signup)

Endpoint:

POST /signup

Request Body (JSON):

{
  "full_name": "Williams Opoku Antwi",
  "email": "williams@example.com",
  "password": "StrongPass123"
}

Example using curl:

curl -X POST https://obala-api.onrender.com/signup \
-H "Content-Type: application/json" \
-d '{
  "full_name": "Williams Opoku Antwi",
  "email": "williams@example.com",
  "password": "StrongPass123"
}'

Response (201):

{
  "message": "Signup successful",
  "api_key": "7e5c4b8b6d9a45b293bf9d7b8b143c0a"
}

5. Send Prompts to OBALA

Endpoint:

POST /obala_chat

Headers:

X-API-Key: YOUR_API_KEY
Content-Type: application/json

Request Body:

{
  "prompt": "K…îk…î…î no y…õ d…õn w…î Twi mu?"
}

Example with curl:

curl -X POST https://obala-api.onrender.com/obala_chat \

-H "Content-Type: application/json" \

8. Example Integration in Python

import requests

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://obala-api.onrender.com"

headers = {"X-API-Key": API_KEY, "Content-Type": "application/json"}
data = {"prompt": "Mepa wo ky…õw, b…î me din."}
